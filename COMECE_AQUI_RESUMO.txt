💡 DICAS PRÁTICAS
═════════════════════════════════════════════════════════════════════════════════

1. PRIMEIRA EXECUÇÃO
   → Copie dados_estoque_exemplo.json para dados_estoque.json
   → Ou comece com dados vazios (será criado automaticamente)

2. DASHBOARD LENTO?
   → Normal com muitos produtos
   → Aumente intervalo em app.js se necessário

3. BUSCA NÃO FUNCIONA?
   → Use navegador moderno (Chrome, Firefox)
   → Limpe cache: Ctrl+Shift+Delete

4. DADOS NÃO SALVAM?
   → Verifique permissões de carpeta
   → Dados são salvos em dados_estoque.json

5. CUSTOMIZAÇÃO
   → CSS: Edite static/css/style.css
   → JavaScript: Edite static/js/app.js
   → HTML: Edite templates/index.html

6. ACESSO REMOTO
   → Para celular na mesma rede:
   → Descubra IP: ipconfig (Windows) ou ifconfig (Linux)
   → Acesse: http://SEU_IP:5000

❓ DÚVIDAS COMUNS
═════════════════════════════════════════════════════════════════════════════════

P: "Python não está no PATH"
R: Reinstale Python e marque "Add Python to PATH"

P: "ModuleNotFoundError: No module named 'flask'"
R: Execute: pip install -r requirements.txt

P: "Conexão recusada em localhost:5000"
R: Verifique se app.py está rodando no terminal

P: "Porta 5000 já está em uso"
R: Edite app.py, procure "port=5000" e mude para "port=5001"

P: "Dados não aparecem"
R: Copie dados_estoque_exemplo.json para dados_estoque.json

P: "Como parar o servidor?"
R: Pressione Ctrl+C no terminal

P: "Funciona no celular?"
R: Sim! Acesse http://SEU_IP_LOCAL:5000
